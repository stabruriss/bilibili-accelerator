# Security Policy

## Supported versions

仅维护仓库 `main` 分支和最新 GitHub Release。

## Reporting a vulnerability

请不要为安全问题创建公开 issue，也不要在报告中附上真实 cookie、token
或完整签名媒体 URL。

优先使用仓库的
[Private vulnerability reporting](https://github.com/stabruriss/bilibili-accelerator/security/advisories/new)
提交报告。请包含：

- 受影响版本。
- 最小复现步骤。
- 预期影响。
- 已完成脱敏的请求或日志。

维护者会尽力确认报告、评估影响并在修复可用后协调披露。
